Step-by-step reproduction (for other teams)

1 — Credentials (never in git)

cp .env.example .env
python3 scripts/setup_env.py   # paste CA + OpenAI keys locally

2 — Auth + hydrate

python3 scripts/auth.py        # M2M first; human login fallback
python3 scripts/fetch_all.py   # workorders, assets, locations → data/
python3 scripts/enrich_public.py  # building-specific NYC Open Data

3 — Local live demo

API_BIND=0.0.0.0 python3 backend/api_server.py
# http://localhost:8787/dashboard/workflow.html

4 — GitHub Pages (submission canvas)

./scripts/sync_pages.sh
git push origin main
# https://<user>.github.io/team12-2402ndave/cover.html

5 — AWS live engine (secrets on server)

./deploy/aws/deploy.sh
# Opens port 8787; .env scp'd to instance; systemd auto-start

# After hackathon:
./deploy/aws/teardown.sh

Design decisions (teaching notes)

  • Backend proxy required — CriticalAsset brief forbids browser-side secrets and CORS direct calls.
  • Pages + AWS split — Public audit trail on git/Pages; live secrets only on ephemeral EC2.
  • Portfolio-only filter — Hides Student Signal: test WOs from demo feed.
  • Confidence labels — AI never claims Verified without evidence (brief pitfall).
  • No MCP — Brief specifies GraphQL backend + workflow UI, not Model Context Protocol.
  • M2M pending — Credential sheet pair returns invalid; human session auth works until event desk activates M2M.
