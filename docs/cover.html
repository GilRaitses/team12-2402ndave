How it was built

  1. Auth — scripts/auth.py: M2M client credentials first; human console session fallback (M2M from credential sheet pending activation at event desk).
  2. Data hydration — scripts/fetch_all.py pulls work orders, assets, locations from CriticalAsset GraphQL into data/*.json.
  3. Public evidence — scripts/enrich_public.py: NYC 311 geospatial query (250m radius at 40.7326568, -73.9844033) + address match + HPD; operational_meaning per hit.
  4. AI structuring — scripts/ai_structure.py: OpenAI gpt-4o-mini JSON contract; regex fallback with UI badge disclosure.
  5. Live API — backend/api_server.py: GET /api/live/workorders, POST /api/structure|record|closure; serves dashboard + docs.
  6. Frontend — Dashboard (filters, map, portfolio-only hygiene) + 6-step workflow (copilot before/after, closure buttons).
  7. AWS deploy — deploy/aws/deploy.sh: EC2 Ubuntu, .env on instance only, systemd team12-api service.
